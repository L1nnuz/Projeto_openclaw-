Relatório de Regras Ativas (HTML5)

Consolidação das regras operacionais, segurança e automações definidas para este ambiente OpenClaw.

Status: Ativo Timezone: UTC Canal principal: Telegram

1) Regras de Identidade e Comunicação

  • Usuário deve ser chamado de Deny.
  • Assistente principal: Alice.
  • Idioma padrão: português direto e objetivo.
  • Tom: charmoso/sensual leve, sem exagero.
  • Não gerar áudio automaticamente sem pedido explícito.

2) Regras de Orquestração Multiagente

  • Lia: triagem e planejamento (prioridades + riscos).
  • Caio: QA e revisão de entrega final.
  • Alice gerencia ambos, consolida e entrega resultado final ao usuário.

3) Regras de Memória em Camadas

  • Camada 0: contexto da tarefa atual.
  • Camada 1: contexto da sessão.
  • Camada 2: memória operacional (templates/processos).
  • Camada 3: memória longa curada (MEMORY.md).
  • Registros diários em memory/YYYY-MM-DD.md.

4) Regra de Website (obrigatória)

  • Sempre que houver pedido de criação/atualização de website, publicar automaticamente no servidor web ativo.
  • Raiz de publicação atual: /var/www/openclaw-site.

5) Regras de Infraestrutura Web

  • Stack ativo: Nginx + PHP-FPM 8.3 + MariaDB + phpMyAdmin.
  • Portas de serviço web: 80 e 8080 (redirect) + 443 (HTTPS).
  • HTTP/8080 redirecionam para HTTPS.
  • phpMyAdmin com restrição de origem (allowlist).
  • Headers de hardening habilitados (HSTS, X-Frame-Options, nosniff, etc.).

6) Regras de Segurança de Host

  • Firewall UFW ativo com política padrão: deny incoming, allow outgoing.
  • Portas liberadas: SSH (22), HTTP (80), HTTP alt (8080), HTTPS (443).
  • Banco de dados permanece local (127.0.0.1:3306).

7) Regras de Automação (Cron)

  • 00:00 UTC diário — Preservação: memória em camadas.
  • 00:00 UTC diário — Preservação: infraestrutura e backup.
  • Job adicional ativo: digest periódico de patch notes (D4/D3/PoE/PoE2).

8) Regras de Versionamento

  • Após alterações relevantes, realizar commit no git do workspace.
  • Sincronizar com GitHub (origin/main) quando solicitado.
Relatório gerado para auditoria rápida das regras operacionais e técnicas do ambiente.
