# Accessibility Checklist (Practical)

## Semantics
- One clear `<h1>` per page
- Logical heading order
- Landmarks present: header/main/footer/nav where applicable

## Keyboard & Focus
- All controls reachable with keyboard
- Visible focus indicator
- No keyboard traps

## Forms & Controls
- Inputs have labels
- Buttons have clear text/icon labels
- Error messages are descriptive

## Visual Accessibility
- Contrast is readable
- Text is not too small
- Status is not conveyed by color alone

## Media & Content
- Meaningful image alt text
- Decorative images with empty alt
- Links are descriptive (avoid generic 'click here')

## Responsive
- Usable at mobile widths (~360px)
- Content does not overflow unexpectedly
- Touch targets are reasonably sized
